# ordr — File Organization CLI

## Overview

`ordr` is a rule-based file organizer CLI. You define rules describing where files should go (by extension, name pattern, date, size), then run `ordr clean` to execute them. It replaces the old "maid" project with a production-quality, Homebrew-distributable binary.

---

## Goals

- Single binary, zero runtime dependencies (Homebrew-friendly)
- Safe by default: preview before any destructive action
- Reversible: undo last clean operation
- Local + global config with clear precedence
- Clean, extensible architecture (adding new rule types is trivial)

## Architecture

Clean Architecture with three layers. Inner layers have zero dependency on outer layers.

```
ordr/
  cmd/
    ordr/
      main.go                    # Entrypoint — wires everything together
  internal/
    cli/                         # Layer 3: CLI (Cobra commands, thin glue)
      root.go
      clean.go
      preview.go
      rules.go
      init.go
      status.go
      undo.go
    core/                        # Layer 1: Domain (pure logic, no I/O)
      rule/
        rule.go                  # Rule types and matching logic
        matcher.go               # Matching engine (extension, pattern, date, size)
      organizer/
        organizer.go             # Orchestrates rule evaluation
        plan.go                  # ExecutionPlan (list of moves, for dry-run)
      config/
        config.go                # Config data structures (domain types)
    infra/                       # Layer 2: Infrastructure (I/O adapters)
      fs/
        filesystem.go            # File system operations (implements core interfaces)
        undo_log.go              # Persists undo log to ~/.ordr/undo.json
      config/
        loader.go                # Finds and loads .ordrrc (walk up + global fallback)
        writer.go                # Writes/updates .ordrrc
  pkg/
    display/
      display.go                 # Formatted output: tables, status, colors
      icons.go                   # Terminal icons / symbols
  .goreleaser.yml                # GoReleaser config for Homebrew tap
  Formula/
    ordr.rb                      # Homebrew formula (generated by GoReleaser)
```

### Dependency flow

```
cli  →  core (organizer, rule, config)
cli  →  infra (fs, config loader)
infra →  core (implements interfaces defined in core)
core  →  nothing (pure Go, stdlib only)
```

## Config Format

Config file: `.ordrrc` (YAML). Searched in this order:
1. `.ordrrc` in current directory (local)
2. `.ordrrc` walking up the directory tree
3. `~/.ordrrc` (global fallback)

Local configs are **merged** with global: local rules take precedence, global rules apply where no local rule matches.

### Schema

```yaml
# ~/.ordrrc
version: 1

rules:
  - name: "PDFs to documents"          # optional, human label
    target: "documents"                # destination directory (relative or absolute)
    match:
      extensions: [".pdf"]             # file extension filter
      pattern: ""                      # regex on filename (optional)
      min_size: ""                     # e.g. "1MB" (optional)
      max_size: ""                     # e.g. "100MB" (optional)
      older_than: ""                   # e.g. "30d", "2w" (optional)
      newer_than: ""                   # e.g. "7d" (optional)
    scope:
      dirs: []                         # limit to specific directories (optional)
                                       # empty = apply everywhere
    options:
      on_conflict: "rename"            # rename | skip | overwrite (default: rename)
      recursive: false                 # recurse into subdirectories

  - name: "Screenshots"
    target: "~/Pictures/Screenshots"
    match:
      pattern: "^Screenshot.*"
      extensions: [".png", ".jpg"]
    scope:
      dirs: ["~/Desktop"]

  - name: "Old large videos"
    target: "~/Archive/Videos"
    match:
      extensions: [".mp4", ".mov", ".mkv"]
      min_size: "500MB"
      older_than: "90d"
```

### Rule matching logic

A file matches a rule when **all** specified matchers pass (AND logic):
- `extensions`: file extension is in the list
- `pattern`: filename (without path) matches the regex
- `min_size` / `max_size`: file size is within range
- `older_than` / `newer_than`: file modification time is within range

If no matcher fields are specified for a rule, the rule is considered invalid and skipped with a warning.

## Commands

### `ordr clean`

Runs all applicable rules on the target directory. Moves matching files to their configured destinations. Creates destination directories if they don't exist.

```
ordr clean [directory] [flags]

Arguments:
  directory    Directory to clean (default: current working directory)

Flags:
  -g, --global          Also run rules that have explicit scope.dirs defined
  -r, --recursive       Recurse into subdirectories
  -n, --dry-run         Preview what would happen without moving anything
      --rule <name>     Run only the rule with this name
  -y, --yes             Skip all confirmation prompts
  -v, --verbose         Show each file evaluated, not just moved files
```

**Behavior:**
1. Load and merge config (local + global)
2. Warn if current directory is a git repo (require `--yes` or prompt)
3. Build an `ExecutionPlan` (list of `Move` operations) without touching the filesystem
4. If `--dry-run`: print the plan and exit
5. Otherwise: print the plan, ask for confirmation (unless `--yes`), then execute
6. Save executed moves to `~/.ordr/undo.json` for undo

**Output (dry-run / preview):**
```
ordr clean --dry-run ~/Desktop

  Preview — 4 files would be moved

  ~/Desktop/invoice_2024.pdf          →  ~/Desktop/documents/invoice_2024.pdf
  ~/Desktop/Screenshot 2024-01-01.png →  ~/Pictures/Screenshots/Screenshot 2024-01-01.png
  ~/Desktop/project_backup.zip        →  ~/Desktop/archives/project_backup.zip
  ~/Desktop/notes.txt                 →  (no matching rule — skipped)

  Run without --dry-run to apply.
```

### `ordr preview`

Alias for `ordr clean --dry-run`. Exists for discoverability.

```
ordr preview [directory] [flags]

  Same flags as clean (except --dry-run which is implicit).
```

---

### `ordr undo`

Reverses the last `ordr clean` operation by moving files back to their original locations.

```
ordr undo [flags]

Flags:
  -y, --yes     Skip confirmation prompt
      --list    Show what the last clean did, without undoing
```

The undo log is stored at `~/.ordr/undo.json`. Each clean session is a stack entry. `ordr undo` pops the most recent entry.

```json
[
  {
    "id": "2024-01-15T14:32:00Z",
    "moves": [
      { "from": "/Users/foo/Desktop/invoice.pdf", "to": "/Users/foo/Desktop/documents/invoice.pdf" }
    ]
  }
]
```

---

### `ordr rules`

Manage the rule configuration interactively and non-interactively.

```
ordr rules [subcommand] [flags]

Subcommands:
  list          Show all rules (global + local, with source)
  add           Interactively add a new rule
  remove <name> Remove a rule by name
  edit <name>   Open the config file in $EDITOR at the rule's line
  test <file>   Show which rules would match a specific file

Flags for list:
  -l, --local   Show only rules from local config
  -g, --global  Show only rules from global config
```

**`ordr rules list` output:**
```
  Rules (5 total)

  Global (~/.ordrrc)
  +-----------------------+---------------------------+--------------------+
  | Name                  | Match                     | Target             |
  +-----------------------+---------------------------+--------------------+
  | PDFs to documents     | ext: .pdf                 | documents/         |
  | Screenshots           | ext: .png .jpg, ^Screenshot| ~/Pictures/...    |
  | Old large videos      | ext: .mp4, >500MB, >90d   | ~/Archive/Videos/  |
  +-----------------------+---------------------------+--------------------+

  Local (./.ordrrc)
  +-----------------------+---------------------------+--------------------+
  | Name                  | Match                     | Target             |
  +-----------------------+---------------------------+--------------------+
  | Project assets        | ext: .svg .png            | assets/            |
  +-----------------------+---------------------------+--------------------+
```

### `ordr init`

Creates a new `.ordrrc` config file.

```
ordr init [flags]

Flags:
  -g, --global    Create global config at ~/.ordrrc (default)
  -l, --local     Create local config in current directory
      --template  Initialize with example rules
```

If a config already exists at the target location, prints its path and exits without overwriting.

---

### `ordr status`

Shows a summary of the current directory relative to the active config.

```
ordr status [directory]
```

**Output:**
```

  Files:   47 total

  Git repo: no
```

## Error Handling

| Situation                     | Behavior                                                  |
|-------------------------------|-----------------------------------------------------------|
| No config found               | Clear message, suggest `ordr init`                        |
| Rule has no matchers          | Warning at startup, rule is skipped                       |
| Destination dir doesn't exist | Created automatically                                     |
| File conflict at destination  | Per-rule `on_conflict` setting (rename/skip/overwrite)    |
| Git repo detected             | Warning + confirmation required (unless `--yes`)          |
| Permission denied             | Error message with path, continue with remaining files    |
| Undo log missing              | Inform user, nothing to undo                              |

## Homebrew Distribution

### Strategy: Custom Tap

Repository: `github.com/<user>/homebrew-tap`

```
brew tap <user>/tap
brew install ordr
```

### GoReleaser (`.goreleaser.yml`)

GoReleaser handles:
- Cross-compilation: `darwin/amd64`, `darwin/arm64`, `linux/amd64`, `linux/arm64`
- Archive creation (`.tar.gz`)
- SHA256 checksums
- Homebrew formula generation and push to tap repo
- GitHub release creation

### Homebrew formula structure

```ruby
class Ordr < Formula
  desc "Rule-based file organizer CLI"
  homepage "https://github.com/<user>/ordr"
  version "1.0.0"

  on_macos do
    if Hardware::CPU.arm?
      url "https://github.com/<user>/ordr/releases/download/v1.0.0/ordr_darwin_arm64.tar.gz"
      sha256 "<hash>"
    else
      url "https://github.com/<user>/ordr/releases/download/v1.0.0/ordr_darwin_amd64.tar.gz"
      sha256 "<hash>"
    end
  end

  def install
    bin.install "ordr"
  end

  test do
    system "#{bin}/ordr", "version"
  end
end
```

## Rule Evaluation Order

1. Local config rules (in order defined)
2. Global config rules (in order defined)

A file is processed by **all** matching rules (one file can be moved to multiple destinations only if rules have different `target` + no conflict). In practice, first matching rule wins per file — subsequent rules that would move the same file are skipped with a note.

```yaml

  push:
    branches: [main]
    paths: ['docs/**']

  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Install & Build
        working-directory: docs
        run: |
          npm ci
          npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/dist
      - uses: actions/deploy-pages@v4
        id: deployment
```
